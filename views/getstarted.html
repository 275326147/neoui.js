Deploy

如果你网络慢或者服务器慢或者没有网络或者作为开发者用于调试, 你可以通过下面内容, 简单的部署项目到本地

安装NodeJs

Build部分使用GruntJs, 所以你懂的


安装python(可选)

使用pythonSimpleHTTPServer做为HTTP服务器, 当然你也可以用nginx等HTTP服务器


获取代码

打开项目托管主页 neoui.js 点击Download下载项目到本地

或者git clone https://github.com/trazyn/neoui.js.git克隆代码到本地


安装依赖

$ cd neoui.js/

$ npm install


启动服务器

安装python后, 在项目根目录执行

$ python -m SimpleHTTPServer


跨域设置

因为部分数据涉及到dev环境服务调用, 所以需要开启浏览器的跨域, 修改浏览器目标参数

"C:\Program Files\Google\Chrome\Application\chrome.exe" --disable-web-security

Develop
Less/CSS

Introduction

样式部分由Less编写, 不建议CSS

思想样式的复用, 可读可维护, 设计原则存在几何形状的组件以ui开头, 允许嵌套进行复用, 命名尽量使用单个单词(从哲学角度讲, 命名越准确, 重用的可能性越小, 比如: .tab-nav-item和.item, 第二个命名就有可能用在tab或者menu等组件中, 所以现在做的是打散命名, 方便后续的重用, 目前还属于打散的阶段)


Style Guide

请保持良好的代码结构, 保证代码的可读, 可维护, 不要写成一坨一坨, 压缩会由脚本执行不需要你去手工压缩

.box{border:1px solid #ddd;height:200px;width:200px;}
.box { border: 1px solid #ddd; height: 200px; width: 200px; .right { ... } .left { ... } }

4格缩进, 1个空格


References

参考资料 Semantic UI, Bootstrap

Javascript

Introduction

组件部分要求无耦合, 简单来讲掰开了还能用, 所以不要过分的抽象, 保持简洁


Style Guide

请参考Google Javascript, Airbnb Javascript, 无硬性要求, 质量第一, 不要出现连作用域, 闭包, 重排重绘等基础都分不清的选手, 不要动不动就来个全局范围的 DOM Query, 或者全局变量的滥用

简单来讲质量第一, 赏心悦目结构良好, 可参考jQuery, Semantic UI等优秀项目

请使用Jshint做静态检查, 参考设置
